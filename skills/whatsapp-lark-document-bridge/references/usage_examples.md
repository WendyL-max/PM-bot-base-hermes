# WhatsApp-Lark Bridge 使用示例

## 基本使用場景

### 1. 項目管理
**場景：** 團隊成員在 WhatsApp 群組中討論項目進度

**示例對話：**
```
用戶A: 今天的會議總結一下
用戶B: 會議要點：
1. 專案進度超前
2. 需要增加測試資源
3. 下周三進行演示

@Lark insert 會議總結：專案進度超前，需要增加測試資源，下周三進行演示 to weekly_report
```

**系統響應：**
```
✅ 內容新增成功！

位置：第8段
段落ID：para_789012
追蹤ID：trace_abc123

✅ 操作完成，無需確認。
```

### 2. 任務分配
**場景：** 主管在 WhatsApp 中分配任務

**示例對話：**
```
主管: 本週任務分配：
- 小明：完成API文檔
- 小華：測試新功能
- 小張：準備演示材料

@Lark todo: 小明完成API文檔
@Lark todo: 小華測試新功能
@Lark todo: 小張準備演示材料
```

**系統響應：**
```
✅ 待辦事項創建成功！

任務：小明完成API文檔
狀態：待處理
創建時間：2024-01-15 15:30
記錄ID：rec_123456

✅ 待辦事項創建成功！

任務：小華測試新功能
狀態：待處理
創建時間：2024-01-15 15:30
記錄ID：rec_789012

✅ 待辦事項創建成功！

任務：小張準備演示材料
狀態：待處理
創建時間：2024-01-15 15:30
記錄ID：rec_345678

✅ 操作完成，無需確認。
```

### 3. 文檔查詢
**場景：** 團隊成員需要查看項目文檔

**示例對話：**
```
新成員: 請問專案規劃文檔在哪裡？
資深成員: @Lark query 專案規劃書
```

**系統響應：**
```
📄 文檔查詢成功！

文檔：專案規劃書
段落數：15
最後更新：2024-01-14 10:45

前3個段落：
1. 專案概述：本專案旨在...
2. 目標：在Q2完成...
3. 時間安排：1月完成設計...

✅ 操作完成，無需確認。
```

### 4. 內容更新
**場景：** 更新過時的信息

**示例對話：**
```
PM: 會議時間有變，請更新
@Lark update para_123456: 會議時間改為週五下午3點
```

**系統響應：**
```
🔄 內容更新成功！

原內容：會議時間：週三上午10點
新內容：會議時間：週五下午3點
變更摘要：更新了會議時間

✅ 操作完成，無需確認。
```

## 錯誤處理示例

### 1. 命令格式錯誤
```
用戶: @Lark 查詢文檔
```
**系統響應：**
```
❌ 命令解析錯誤

錯誤：未知命令 "查詢文檔"
建議：使用 @Lark help 查看可用命令

✅ 操作完成，無需確認。
```

### 2. 文檔不存在
```
用戶: @Lark query 不存在的文檔
```
**系統響應：**
```
❌ 文檔查找失敗

錯誤：找不到文檔 "不存在的文檔"
建議：檢查文檔名稱或聯繫管理員

✅ 操作完成，無需確認。
```

### 3. 權限不足
```
用戶: @Lark insert 機密信息 to restricted_doc
```
**系統響應：**
```
❌ 權限不足

錯誤：您沒有權限修改此文檔
建議：聯繫文檔所有者獲取權限

✅ 操作完成，無需確認。
```

### 4. 網絡問題
```
用戶: @Lark query project_doc
```
**系統響應：**
```
❌ 網絡錯誤

錯誤：無法連接到 Lark 服務器
建議：檢查網絡連接後重試

✅ 操作完成，無需確認。
```

## 最佳實踐

### 1. 命名規範
- 文檔名稱使用英文或拼音，避免特殊字符
- 任務描述簡潔明確
- 使用一致的日期格式

### 2. 命令格式
- 每個命令單獨一行
- 使用正確的語法
- 必要時添加參數說明

### 3. 錯誤處理
- 檢查命令格式再發送
- 確認文檔存在再操作
- 遇到錯誤查看錯誤信息

### 4. 效率技巧
- 使用批量操作提高效率
- 預先準備好命令模板
- 定期清理無效命令

## 故障演練

### 場景1：命令無響應
```
用戶: @Lark query test_doc
（無響應）
```
**解決步驟：**
1. 檢查網絡連接
2. 確認服務運行狀態
3. 查看日誌文件
4. 聯繫管理員

### 場景2：響應錯誤
```
用戶: @Lark insert test
系統: ❌ 未知錯誤
```
**解決步驟：**
1. 記錄錯誤信息
2. 嘗試簡單命令測試
3. 檢查配置文件
4. 重啟服務

### 場景3：性能問題
```
用戶: @Lark query large_doc
（響應緩慢）
```
**解決步驟：**
1. 檢查服務器負載
2. 優化文檔結構
3. 使用緩存
4. 分頁查詢
